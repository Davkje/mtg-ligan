# MTG Commander League — Project Overview

A fullstack web app for a Magic: The Gathering Commander league. Supports 3–5 player games.

## Stack
- **Next.js 16.2.3** (App Router) — see AGENTS.md for breaking-change notes
- **React 19** with React Compiler enabled
- **Supabase** — Postgres DB + anon key client
- **Tailwind CSS v4** — CSS-first, no config file; use `@theme` in globals.css
- **Framer Motion** — mobile nav drawer animation
- **TypeScript 5**

## Environment Variables
```
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
ADMIN_PASSWORD=      # protects /register, editing matches, adding/renaming players
MASTER_PASSWORD=     # protects deleting matches and deleting players
```

## Pages
| Route | Description |
|-------|-------------|
| `/` | Leaderboard + last 5 matches feed |
| `/register` | Password-protected match registration form |
| `/history` | Full match history; edit (admin) / delete (master) |
| `/players` | All players list + add player (admin) |
| `/player/[id]` | Player profile + achievements + rename (admin) / delete (master) |
| `/match/[id]` | Match detail page |
| `/about` | Rules, scoring table, achievement descriptions |

## Point System
Points scale with player count:

| Placement | 3p | 4p | 5p |
|-----------|----|----|-----|
| 1st | 80 | 100 | 120 |
| 2nd | 60 | 75 | 90 |
| 3rd | 40 | 50 | 60 |
| 4th | — | 25 | 30 |
| 5th | — | — | 15 |

Scoring is always calculated dynamically via `getPoints(playerCount, placement)` — never stored in DB.

## Database Schema
- `players` — id, name
- `matches` — id, played_at (date), notes (text, nullable)
- `match_results` — id, match_id, player_id, placement (1–5)

## Admin Auth
- `ADMIN_PASSWORD` → cookie `admin_session` via `POST /api/auth` — protects creating/editing matches, adding/renaming players
- `MASTER_PASSWORD` → cookie `master_session` via `POST /api/master-auth` — protects deleting matches (`DELETE /api/matches/[id]`) and deleting players (`DELETE /api/players/[id]`)

## API Routes
- `POST /api/auth` — sets admin_session cookie
- `POST /api/master-auth` — sets master_session cookie
- `GET/POST /api/players` — list all / add new (admin)
- `PUT/DELETE /api/players/[id]` — rename (admin) / delete (master, blocked if has match history)
- `POST /api/matches` — create match (admin)
- `PUT/DELETE /api/matches/[id]` — edit (admin) / delete (master)

## Achievements (calculated dynamically)
- 🏆 "One of Every Kind" — received all placements at least once
- 😈 "Fallen from Grace" — won 2+ in a row, then came last
- 💪 "The Underdog" — won after 3+ consecutive losses
- 🔒 "Consistency is Key" — no last-place finish in 5 matches in a row
- ⚡ "Hot Streak" — won 3 matches in a row

## Key Patterns
- All data pages use `export const dynamic = "force-dynamic"` to always fetch fresh from Supabase
- Supabase TS client resolves joined queries as `never` — use `(supabase.from("table") as any).insert(...)` for writes, and explicit `as Array<...>` casts on reads
- `RouteContext<'/path/[id]'>` is a global type (generated by Next.js into `.next/types/`) — do NOT import it, just use it
- Achievements use `GameResult { placement, wasLast }` — call `toChronological()` before `computeAchievements()`
- Mobile nav: `MobileNav` client component (Framer Motion drawer, `sm:hidden`); desktop nav is `hidden sm:flex` in layout

## Key Next.js 16 Gotchas
- `params` and `searchParams` in page/layout components are **Promises** — always `await` them
- `cookies()` and `headers()` from `next/headers` are **async** — always `await` them
- Cache revalidation: use `revalidatePath` / `revalidateTag` from `next/cache`; `refresh()` for router-only refresh
- `React.FormEvent` is deprecated in React 19 — use `React.SyntheticEvent`
